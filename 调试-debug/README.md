# 调试 - debug

## 简述

node 断点调试，目前主要由三种方式：

+   node 内置调试工具
+   IDE debug：vscode、webstorm 等
+   node-inspector

三者本质上差不多，更多区别在用户体验。

## node 内置调试工具

测试脚本 `index.js` 代码如下：

```js
function foo() {
    const c = 3;
    return c;
}

const a = 1;
console.log(a);

const b = 2;
console.log(b);

foo();
console.log('done');
```

+   进入调试模式：`node debug index.js`

    +   在第一行断点：默认就是第一行
    +   在第三行断点
        +   方式一：通过 debugger。在命令行交互中输入：`cont` 跳到第一个 debugger 断点
        +   方式二：通过 `sb(lineNumber)`。如：`sb(3)`
    +   执行下一步：通过 `next` 命令
    +   调到下一个断点：`cont`
    +   查看某个变量的值：输入 `repl`，然后输入变量名，就可以看到变量的值，然后按 `ctrl + c` 退出
